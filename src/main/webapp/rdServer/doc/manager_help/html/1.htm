데몬의 구동 요건

데몬은 Installation Guide에서 기술한 바와 같이 WAS와 함께 쓰이거나 또는 독립적으로 구동할 수 있다. (참고 Installation Guide 의 1. RDServer 소개)

데몬은 기본적으로 관리자를 연결시킬 수 있는 관리포트를 필요로 한다. 포트의 지정은 다음과 같이 바꿔줄 수가 있다.

독립적으로 구동을 시킬 경우

Rdserver/bin 밑에 있는 startup.bat(Windows 계열)이나 startup.sh(Unix/Linux 계열)파일을 열어 manager.port의 포트번호를 변경해 준다.

WAS와 함께 구동을 시킬 경우

RDServer/rdagent.jsp파일을 열어

private static int managerport =8089;

에 기본으로 정의된 포트번호를 바꾸어 주면 된다.

WAS서버를 이용해서 구동시킬때는 (서블릿 버전) 별도의 startup.sh 스크립트 실행없이 WAS 구동 후 다음과 같이 호출 시 자동 구동된다.

-Viewer 또는 Editor 에서 처음 호출 시

-웹브라우저에서 http://serverip:port/RDServer/rdagent.jsp 호출 시

참고) 서블릿 버전을 RDServer/bin/startup.sh 또는 startup.bat 실행 후 클라이언트에서 호출하게 되면 서버 관리포트에 대한 BindException 이 발생 할수 있으므로, 서블릿버전으로 운영시에는 스크립트를 실행하지 말아야 한다.

데몬 서버 구동

인스톨러를 이용하여 설치를 마치고 곧바로 자동 구동이 가능하다.

서블릿 버전 서버 구동은

수동 구동은 다음과 같이 실행하면 된다. (기본 포트: 8282)

Windows 계열: RDServer/bin/startup.bat

> startup.bat

유닉스/리눅스 계열: RDServer/bin/startup.sh 를 chmod +x 하여 실행 모드로 수정 후 실행

> startup.sh

 

그림 (1) 정상적인 서버 구동 예

그림 (1)은 서버가 아무 에러나 이상 없이 정상적으로 구동됐을 때의 예를 보여준다.

>>서버 구동시 옵션

startup.bat 또는 startup.sh 에 옵션 처리 가능

:java -Dfile.encoding=EUC_KR -classpath "%RDServLib%" m2soft.rdsystem.server.Server debug server.port=8282 manager.port=8089 oencoding=false userpool=false runschedule=false

debug : 서버 구동 및 실행시 콘솔창에 세부 수행 상태 출력 여부

server.port : 서버 구동 포트

manager.port : 서버 매니저 포트

oencoding: 데이터 패치시 encoding 여부 (true,false) : true à 사용자 환경이  한글O.S면서, 데이터베이스 charset 이 UTF-8 이나 ASCII(8859_1) 로 셋팅되어 있는경우 사용한다. db.properteis 의 charset 에 데이터베이스 charset 을 정의하여 주게 되어 있다

userpool: 사용자 정의 커넥션 풀을 사용할 경우 (기본 false)

runschedule : 스케줄링 기능을 사용할 경우 (기본 false)

그림 (2) 데이터베이스 > 데이터베이스 접속창

그림 (1)의 예에서는 RDServer가 8282에 열려있는 것을 볼 수 있다. 해당 Server의 IP와 포트번호를 그림 (2)에서 처럼 RDAgent경로에 적어주고 이용하고 싶은 서비스를 사용할 권한이 있는 사용자 아이디와 패스워드를 친다. 사용자 계정을 성공적으로 체크 완료 시에는 “등록된 서비스명”에 사용할 수 있는 서비스의 리스트가 뜨는 것을 볼 수 있다. 여기에서 사용하고 싶은 서비스를 선택하고 접속을 클릭하면 그림 (3)과 같은 메시지가 나온다.

참고: 사용자 아이디 추가는 9.시스템관리-보고서 보안 참조

그림 (4) 정상적으로 서버를 종료시킬 때의 예

 

유닉스/리눅스 계열: RDServer/bin/shutdown.sh 를 chmod +x 하여 실행 모드로 수정 후 실행

> shutdown.sh

 

위와 같이 shutdown.sh을 실행시키면 구동 중인 서버가 종료된다.
