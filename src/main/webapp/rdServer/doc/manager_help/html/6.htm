<실행전점검사항>

메일통지기능을사용하기위해 RDServer  IP 가스팸 IP로등록되어있지않은지확인

RDServer에 db.properties,rd.properties 파일이 /RDServer/conf 아래에있는지확인하고, 데이터베이스에대한설정이맞는지확인

JDK1.3 이상(이하버전에서는스케줄링기능을사용할수없음)

 

<처리과정>

스케줄링작업은 RDServer 관리자를통해등록하고, 처리는서버에작동중인스케줄링서버가수행한다.

 

스케줄링서버시작

 

서블릿버전: rdagent.jsp 최초접속시자동 Start (rdagent.jsp)

private static boolean schedule = true;

데몬버전: 서버구동시자동 Start (startup.bat)

runschedule=true

최대입력값은 40자이다.

예)/myfolder/myDoc

>>myfolder 라는디렉토리생성후 myDoc….html 파일을생성

실행파라미터참조 index: 스케줄링결과파일명에파라미터값을추가하여생성하고싶을때 /rp 파라미터값에서참조할파라미터 index 를입력한다. Index 는 1부터시작하며, 최대입력값은 99 이다

실행파라미터를 /rp [1] [myid] 로줄경우실행파라미터참조 index를 2로입력하면결과파일생성은날짜_myid.html 와같은형태로생성

메일발송여부:

스케줄링완료후메일통지여부를선택한후 메일수신자를입력한다. 받는사람이복수인경우세미콜론(;)으로열거한다.
최대입력값은 512자이다.

만약에메일리스트를 jsp 또는 asp를호출하여가져오고싶으면, 다음의예처럼 URL 를입력한다.

예) http://ip:port/schedulemaillist.jsp

출력결과는직접입력할경우와같은형태의결과여야한다

jsp 구문)

out.print("shkim@m2soft.co.kr;meene@hanmail.net;")

asp 구문)

Response.Write("shkim@m2soft.co.kr;meene@hanmail.net ")

<참고사항> 실행파라미터에다중파라미터값을입력한경우세미콜론(; ) 구분자로수신자입력란에주소를입력한다. 그러면 /rp 입력순서에따라차례로매핑되어메일이발송된다. 실행파라미터에대응되는메일입력값이없을경우메일발송되지않음.

 

메일발송형식: 메일발송시내용을 HTML 형식으로 embed 시킬때는 HTML를선택하고, 스케줄링완료파일위치만링크시킬때는 URL LINK를선택한다.

- URL LINK 선택시스케줄링메일통지양식(HTML) 작성하기참조

예) HTML 형식으로메일을보낸경우 (기준: Outlook  express 이며,

내용에 RD ActiveX 가바로보이는형태)

 

<참고사항>

* HTML 형식으로보낼때는사용자가사용한메일프로그램이낮은보안수준옵션을가져야한다

예) Outlook expresss 경우:

           도구/옵션/보안-인터넷영역(보안수준은낮고, 기능성은높음)

* 스케줄링이제대로시행되지않을때스케줄링/서비스제어메뉴에스케줄링을재시작또는일시중지시킬수있다.

* 스케줄링은시작되었으나쿼리및데이터의양에따라데이터파일이즉시생성되지않을수도있으며, 서버매니저의로그/스케줄링로그보기를참조하여스케줄링관련진행상태를파악할수있다

스케줄링메일발송

 

스케줄링메일발송은이미스케줄링이끝난파일을선택해서메일을보낼수

있는기능이다.  $RDServerHome/ocxhtm 아래에있는 html 파일들을 

정의된메일아이디로메일을발송하게된다 .

 

 

 

소스파일: 발송해야할정보가담긴파일경로를입력한다. 입력제한

길이는128자이다. 텍스트파일(서버의절대경로) 또는 url 로정의해줄수있다.

 

소스파일위치입력예)

Windows 계열: C:/m.txt

linux,unix 계열 : /usr/local/RDServer/m.txt

 

소스파일의내용은발송해야할대상파일위치와메일수신자를 , 로구분하여정의한다. 건당구분자는리턴캐릭터정의해준다.

 

소스파일의내용예) 

 

admin/TEST2004120111000011_VINET.html,VINET@m2soft.com

admin/TEST2004120111000011_AROUT.html,hssong@hotmail.com

:

 

$RDServerHome/ocxhtm 디렉토리의파일리스트

 

 

발송제목: 발송시제목을입력한다. 제목최대길이는 30자내외이다

실행: 소스파일을읽어, 메일발송을시작한다서버에기존에진행중인작업이있으면실행하지않는다.

실행후결과는서버의처리시간이있으므로바로결과가오지않을수도있으며, 그럴경우로그보기-스케줄링로그메뉴를참고한다

중지: 현재진행중인메일발송작업을취소한다. 발송되지않은작업위치부터메일을발송하지않는다

모니터링시작: 발송되는과정(성공,실패)을실시간모니터링한다 (실행이처음으로이루어질때기본값은모니터링시작)

모니터링중지: 모니터링을중지한다.

 

스케줄링서버중지

 

          

서블릿버전:  WAS 중지시자동종료

데몬버전: 서버정지시자동종료
