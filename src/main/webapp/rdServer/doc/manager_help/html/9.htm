업데이트

RDServer를 실시간으로 업데이트가 가능하다. 즉 설치되어있는 RDServer에서 최신의 RDServer의 파일 경로(http://m2soft.co.kr/report/RDServerUpdate/...)를 찾아서 업데이트한다. 파일을 받기 전 백업하므로 파일이 깨졌을 때에도 안전하게 복구가 가능하다. 업데이트에 관한 환경설정은 메뉴에서 환경설정>>업데이트 및 백업 클릭 후 업데이트 탭에서 설정 가능하다. 주의사항) 업데이트 종료 후 RDServer를 다시 리스타트 과정을 거쳐야 하며, 각각의 스텝은 아래와 같다.

 

스텝 1. 업데이트에 관련된 일반적인 사항을 나타낸다.

그림 (1)

 

스텝 2. 현재의 RDServer와 최신의 RDServer를 비교하여 나타낸 것으로 서버를 빌드한 버젼과 날짜를 보여주며 최신의 RDServer가 버전이 높으면 새로운 기능과 참고사항을 보여주고 업데이트를 계속할 수 있다. 최신의 버전과 같거나 높을 경우 업데이트는 이 스텝에서 멈추게 된다.

 서버관리 테이블

 

RDServer서버는 관리를 위하여 5개의 테이블을 같게 되는데 이 테이블들에 대한 정보를 순번, 이름, 필드 수, 레코드 수, 테이블 정보로 보여지게 되며, 테이블들을 삭제 및 초기화할 수 있다. 테이블의 각 기능은 아래와 같다.

- SCUSER : 관리자로 로그인하기 사용자 테이블

- SCDIRECTORY : 스케줄링 파일의 위치를 가지고 있는 테이블

- SCFILEDESC : 스케줄링 파일의 상세정보

- SCHEDULE : 스케줄링 관리를 위한 테이블

- SCHEDULELIST : 스케줄링 관리를 위한 테이블

 

그림 (1)

<테이블 삭제>

테이블 삭제 버튼을 클릭하게 되면 5개의 테이블이 모두 삭제되며, 기존의 정보는 모두 사라지게 된다. 아래의 텍스트 창에는 테이블의 삭제와 초기화가 되는 리턴값을 보여주는 것으로 이것을 확인하면 실제 테이블이 생성되는지 또는 생성시 실패했는지를 알 수 있다.

그림 (3)

 

<초기화>

초기화는 테이블의 내용이 모두 지워지는 것으로 기존의 자료는 모두 지워지게 된다. 그림 (5)는 테이블이 초기화된 것을 보여주는데 레코드의 건수가 모두 0인 것을 알 수 있다.

 

보고서 보안

보고서 보안은 서버에 등록된 사용자만을 데이터베이스에 접근할 수 있도록 하는 기능이다.

즉,클라이언트(에디터,뷰어,OCX)에서 접속할 때 아이디와 패스워드를 요구하며, 사용자가 사용할수 있는 데이터베이스 서비스명(JDBC)을 돌려주게 된다.

이 메뉴를 통해 등록된 사용자만이 보고서를 작성할 수 있다.

그룹 추가

기본 그룹은 Users 와 Administrator 로 나누어 지면 새로운 그룹을 추가하거나 삭제할 수 있다. 사용자를 추가할 때 반드시 그룹 중 하나를 선택해야 하며, 그 그룹이 사용할수 있는 서비스명을 추가 시켜주어야 한다. 서비스명은 환경설정-JDBC 서비스에 정의된 이름들이다.

서비스명 추가( Service Name(s) ) : 그룹을 선택한 후 서비스명의 추가 버튼을 눌러 추가 가능한 서비스명이 나오면 선택하여 추가한다.

사용자 추가

사용자 탭을 눌러 새로운 사용자의 아이디와 패스워드, 그룹을 선택한 후 추가버튼을 누른다. 사용자 추가는 서버 restart 없이 바로 적용 가능하며 그룹을 Administrator 또는 Users로 선택하면 관리자 로그인의 계정으로 사용할 수 있습니다.
