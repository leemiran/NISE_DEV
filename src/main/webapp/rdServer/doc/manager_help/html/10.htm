환경설정

환경설정의기본적인메뉴는다음과같다.

JDBC 서비스

어떠한풀을사용하느냐에따라설정화면이달라진다. 공통으로설정하는옵션은다음과같다.

암호화저장 : 이체크박스의체크상태유무는곧현재서버에저장되어있는 db.properties 파일의암호화유무를나타낸다. 만약 JDBC 서비스화면로딩시체크가되어있었다면, 현재 db.properties는암호화상태이고, 이것을사용자가언체크후확인을누르면서버에저장되어있는파일은복호화되어저장이된다.

로그파일경로 : 데이터베이스연결로그. DB접속및끊기에대한로그가여기에정의되어있는위치의파일명에저장이된다. 파일이존재하지않을경우새로이생성을한다.

풀사용 :어떠한풀을사용할것인지를결정한다. WAS에서제공하는풀이나 RDServer의풀을사용할수있다.

 

RDServer풀

기본설명

RD에서제공하는커낵션풀을썼을때의 RDServer풀설정화면이다. db.properties에서정의된서버스리스트는하나의서비스가하나의탭으로정의되어표시된다. 사용자는해당서비스탭을선택함으로써각각의설정을바꾸거나지정할수있다.

 

 

그림 (1) RDServer풀사용시 properties설정화면

그림 (2) 추가버튼을눌렀을때

 

그림(3) 기존입력된정보가변경되지않았는데확인버튼을누른경우

그림(4) Pool 정보수정시커넥션을바로리로딩시킬지의여부를나타냄

 

그림(5) 새로운서비스를추가한후확인(즉시적용)을수행하였으나추가된서비스의등록정보가틀릴경우.

 

항목설명

URL : 데이터베이스연결정보(DB서버, 포트)을설정한다.

사용자 : DB계정아이디을설정한다.

암호 : DB 계정암호를설정한다.

최대커낵션수 : 커낵션최대연결수을설정한다.

문자셋 : DB에서사용하는문자셋(charset)을설정한다.

 

 

추가

그림(2)는추가버튼을눌렀을때나오는설정창을보여준다. 여기에서사용자는추가될새로운서비스의명을입력하면된다. 이미추가되어있는동일한서비스명을입력했을경우 “새로운서비스” 창은닫히질않고다시프롬트된다.

 

새로운서비스를추가하였을때기본적으로 “해당서비스로아직연결이되지않았습니다”라는메시지가뜬다. 이상태에서사용자는새로추가한서비스의등록정보를적어넣고확인을클릭하면즉시적용하여로딩할것인지재구동시적용할것인지를선택하게된다.

즉시적용을선택하였다면 RDServer는새로운서비스의등록정보를이용하여커낵션을테스팅하고실패했을경우그원인을연결상태에서보여준다.

 

재구동시적용을선택했을경우, 해당서비스의탭은빨간색으로표시된다. 연결상태창은해당서비스가존재하지않는다는메시지를보여줄것이다. (아직업데이트가되지않았음으로당연하다.) 사용자가다시확인을선택하여즉시적용을선택하거나서버를재구동하여메니져로접속하였을경우 RDServer 는해당서비스를테스팅하여그결과여부를연결상태에서보여준다.

 

확인

확인을누를시에는그림(3) 과 (4)와같이두가지메시지가나올수가있다. 사용자가새로운데이터를입력하지않았을경우그림 (3)처럼업데이트를필요로하지않습니다. 라는메시지가나오고만약새로운데이터가입력되었을경우는그림(4)와같은 “Connection Reloading”이란화면이뜬다.

 

“Connection Reloading”에는 “서버재구동시적용”과 “즉시적용” 두가지옵션이있다. 서버재구동시적용은서버를다시리스타트했을때 DB와의풀설정을변경하는것이고, 즉시적용은 DB와의모든커낵션을끊고새로이풀을생성하여접속하는방법이다.

 

확인버튼을누름과동시에선택된옵션이실행된다.

WAS풀

서버가 WAS 위에서수행되는서블릿버전인경우사용할수있다

서버파일의 RDServer/rdagent.jsp 에 WAS 풀사용값이 true 로설정된경우적용된다.

:

private static boolean waspool = true;

그림 (5) WAS풀사용시 properties 설정화면

만약현재 RDServer가 WAS의풀을사용할때에는 JDBC 서비스선택시그림(5)와같은설정화면이나온다.

화면에표시된바와같이 db.properties의 waspools 키에정의되어있는 JNDI 풀이름들의리스트탭으로보여준다.

db.properties에서각각의 JNDI풀명은스페이스(“ “)을구분자로설정되어있다.

예)  waspools=jdbc/MyOraclePoolDB jdbc/MyOraclePoolDB1 jdbc/MyOraclePoolDB2

 

그림 (6) 추가를선택했을때

 

그림 (7) 새로운 WAS풀명입력후확인시나타나는화면

 

그림(6)은 WAS풀설정화면에서추가버튼을눌렀을경우뜨는 “새로운 WAS풀” 설정창을보여준다. 여기에추가될새로운 WAS풀명을입력하고확인을눌러주면된다. RDServer풀설정화면의추가버튼과마찬가지로이미리스트에있는 WAS 풀명을쓸경우 “새로운 WAS풀” 창이다시프롬트된다.

 

그림(7)은새로운 WAS풀을추가하고나서의다음설정창과탭의변화를보여준다. 그림(6)에서우리는 jdbc/MySqlPoolDB라는 WAS풀을추가하였기때문에이풀명을가진탭이새로이생성된것을그림(7)에서확인할수가있다.

“Connection Reloading”에는 “서버재구동시적용”과 “즉시적용” 두가지옵션이있다. 서버재구동시적용은서버를다시리스타트했을때 DB와의풀설정을변경하는것이고, 즉시적용은 DB와의모든커낵션을끊고새로이풀을생성하여접속하는방법이다.

그림(8) 삭제를선택했을때

 

그림(8)은삭제를선택했을때의 “WAS 풀삭제”창이다. 여기에서선택된 WAS 풀이란현제보여지는탭의명을가르킨다.

“예”를선택하면선택된 WAS풀이삭제되면서그림(7)과같이적용시기를묻는 Connection Reloading창이다시뜬다.

스케줄링및기타환경

상단에보면암호화저장체크박스가있다. 로딩시체크의유무는현재파일의암호화상태를나타내며, 사용자가체크상태를바꾸어주고확인을누름으로써서버에저장되어있는 rd.properties파일을암호화및복호화시킬수가있다.

숫자를제외한각종패스및입력길이제한은각각 200자내외이다.

에러로그경로: 스케줄링수행시에러로그를기록할파일명. 에러로그파일의절대위치를설정한다.

MRD 파일업로드경로: RDServer에 mrd을업로딩하는위치

문서실행후생성된데이타파일의위치: 스케줄링후보고서의데이터파일위치

메일발송양식 HTML: 스케줄링후메일을보낼때의기본양식이되는 HTML.의위치

CAB 버전: RDServer/rdviewer.cab 의버전 (cab 파일을압축해제프로그램을이용하여풀면 rdviewer40.inf 파일에 FileVersion= key 를찾아보면알수있다. 콤마(,) 및숫자만입력가능,길이제한 15자)

스케줄

WAS풀사용: RDServer가커낵션을생성할때 WAS풀을사용할것인지 RDServer풀을사용할것인지를정의하는필드이다. 이옵션이변함에따라재로딩시다음필드가 JDBC 서비스명/JNDI풀명으로바뀌는것을볼수있다.

JDBC 서비스명: WAS풀사용옵션을바꾸고재로딩을하면이필드의레벨이따라서바뀌는것을볼수가있다. 해당풀의종류에따라 JDBC 서비스명내지 WAS Pool명을적어주면된다.

한글인코딩여부: 스케줄링서버가캐랙터셋을지원하지못할경우이옵션을 TRUE로맞춤으로써지원가능케한다.

스케줄링데이터파일저장구분자: 스케줄링의테이타파일에서쓰일구분자를지정해준다.

언어: 언어를지정해준다.

사용자정의문자셋: 서버의언어캐랙터셋을입력한다.

스케줄데이타파일포멧: 스케줄링된데이터파일명의포멧을지정해준다. TRUE일경우는 yyyyMMdd, FALSE일경우는 yyyyMMddhhmmss+serial 로저장이된다.

DM 서버 IP: 파일업로딩을할때 DM서버의 IP주소를입력한다. 현재 IP 로만등록가능하며, 입력제한길이는 . 을포함하여 15자이다

DM 포트: DM 서버의포트를입력한다. 입력제한길이는 15자이다

DM Context Path: DM 서버가아닌웹서버디렉토리로업로드할경우 DM 서버의 Context Path을입력한다. 예) /RDServerdm

DM 웹서비스포트: DM 서버가아닌웹서버디렉토리로업로드할경우의웹서비스포트를입력한다. 입력제한길이는 15자이다

* 참고) Context Path와웹서비스포트가설정되어있지않을경우 DM 서버에서스케줄링파일을서비스한다.

 

로그

상단에보면암호화저장체크박스가있다. 로딩시체크의유무는현재파일의암호화상태를나타내며, 사용자가체크상태를바꾸어주고확인을누름으로써서버에저장되어있는 log.properties파일을암호화및복호화시킬수가있다.

각로그에대한위치를설정할수있으며, 수정후적용은서버 restart 가능하다

 

에러로그

환경설정메뉴의로그를선택하여에러로그에대한위치및로그파일의최대파일의수, 크기를설정할수있다

 

 

에러로그저장위치: 로그가쓰여질디렉토리및파일명을절대경로로입력한다

에러로그레벨: 에러를기록하는기본 level를지정한다.

(옵션 : debug > info > error > none )

최대로그파일사이즈:  로그파일당크기.

최대로그파일수 : 로그파일의생성개수.

로그파일갱신기간: 서블릿버전에대한모니터링시필요한옵션으로로그를보는 Refresh 시간을초단위로입력할수있다.

에러출력방향: 에러를 File 또는 console로보낼지를설정한다. 콘솔을선택하면오류가표준출력으로디스플레이된다.

 

보고서사용자로그

          

 

로그기록여부:  기록여부설정

로그레벨: File(0) 또는 Database(1) 에기록할지를설정한다

새로고침기간: 모니터링간격을초단위로설정한다(서블릿버전만해당)

로그파일위치: 모니터링로그파일위치

로그파일최대사이즈 : 로그파일의파일크기

로그파일최대파일수: 로그파일파일의개수 (초과할경우순차적으로 overwrite 됨)

서비스이름(JDBC): 로그기록을 Database(1)에할경우사용할 JDBC 서비스를선택한다. 서비스명은환경설정-보고서-JDBC 서비스참고.

업데이트백업의최대저장수: 업데이트에대한백업을몇회실행할것인지에대한설정을한다.

SAP

SAP R/3 의 RFC나테이블를이용하여보고서를작성하고싶다면, 문서디자인시에 SAP GUI 클라이언트개발자 PC에설치하여문서를작성하여야한다.

RDServer는런타임시에 SAP 를연결하여데이터패치기능을제공한다(서버시작시 sapjco 관련클래스와라이브러리가클래스패스에추가되어야함).

다음은 SAP 서버에연결하기위한서버측환경설정을위해제공하는화면이다.

상단에보면암호화저장체크박스가있다. 로딩시체크의유무는현재파일의암호화상태를나타내며, 사용자가체크상태를바꾸어주고확인을누름으로써서버에저장되어있는 sap.properties파일을암호화및복호화시킬수가있다. 로그파일경로는 sap에관련된로그파일을저장할경로를나타낸다.

 

사용자: SAP Server에등록된사용자아이디.

암호: SAP Server에등록된사용자의암호.

클라이언트: 설치된 SAP Client의 ID.

언어: SAP Server가사용하는언어.

최대커낵션수: 최대한의커낵션개수.

시스템번호: 시스템번호.

호스트: 호스트명또는호스트 IP을설정한다.

 

이노룰스

RBMS(Rule Base Management System) 을연결하여보고서를작성하는방법을제공한다. 다음은이노룰스시스템에접근하기위한서버측환경설정화면이다.

상단에보면암호화저장체크박스가있다. 로딩시체크의유무는현재파일의암호화상태를나타내며, 사용자가체크상태를바꾸어주고확인을누름으로써서버에저장되어있는 innorules.properties파일을암호화및복호화시킬수가있다.

 

 

서버 IP: 이노룰스서버의 IP.

서버포트: 이노룰스서버의포트번호.

인코딩여부: 인코딩옵션.

문자셋: 이노룰스서버에세팅되어있는문자셋.
