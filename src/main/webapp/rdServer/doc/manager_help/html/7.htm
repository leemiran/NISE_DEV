모든 모니터링 화면 창들은 줌인이 가능하다. 줌인은 해당 탭을 더블클릭 하거나 해당창의 타이틀 바를 더블클릭하면 된다.

메모리 사용량

메모리 사용량은 현재 RDServer가 구동중인 서버의 JVM의 메모리 사용량을 실시간 그래프로 나타내준다.

그림 (2)

메모리 사용량 모니터링은 위의 그림 (1)에서 표시된 바와 같이 툴바에서도 액세스가 가능하다.

다음 그림은 메모리 사용량 모니터링을 선택했을 때의 기본화면 이다.

 

Information Box 설명

최대 메모리 (Max Memory)

JVM의 최대 메모리 크기를 나타낸다.

전체 메모리 (Available Memory)

JVM의 상위에서 돌아가는 모든 프로그램들을 위해 확보해 놓은 전체 메모리의 크기를 나타낸다. 만약 더 많은 메모리 공간이 필요 시에는 최대메모리에서 필요한 분량을 가져 다 쓴다. 전체 메모리의 크기는 가져온 분량만큼 증가한다.

사용 메모리 (Used Memory)

JVM위에서 돌아가는 프로그램들이 현재 사용하고 있는 메모리의 분량을 나타낸다.

가용 메모리 (Free Memory)

전체 메모리에서 사용메모리의 값을 뺀 가용 메모리를 나타낸다.

그래프 설명

X축

전체 메모리를 100%로 잡아 현재 사용중인 메모리와 여유 분으로 남아있는 메모리의 그래프를 실시간으로 보여준다.

 

Y축

시간의 흘려감을 표시해 준다. 그래프는 오른쪽에서 왼쪽으로 움직인다.

현 사용자 모니터링

다음은 현 동시 사용자의 숫자를 실시간으로 보여주는 모니터링 화면이다.

 

상태 블록 스택

오른쪽의 블록 스택은 즉각적인 현 사용자수를 시각적으로 보여준다.

사용자수가 25명이나 그 이하일 경우에는 한 개의 노란색 블록이 한명의 사용자를 나타내고 그 이상이면 한 개의 오렌지색 블록이 10명의 사용자를 나타낸다. 그래프선의 색깔은 25명이나 그 이하는 빨간색으로 25명 이상은 파란색으로 현 동시 사용자수를 나타낸다. 상태 블록 스택의 하위에는 정확한 숫자가 표시된다.

 

그래프

X축은 시간의 흐름을 나타낸다. 1초에 한번씩 데이터를 업데이트하여 표시한다.

Y축은 현 동시 접속 사용자수를 나타낸다. 그 단위는 오른쪽 상단에 있는 레벨에서 확인할 수 있다. 기본적인 단위는 일 단위이다. 25명 이상일 경우는 십 단위로 바뀌어 표시된다.

그래프의 하단에는 시간의 흐름을 나타난다. 타이머는 사용자가 로그인후 현 사용자수 모니터링을 선택하고 뷰(View)를 하는 동안의 시간을 잰다.

 

 

 

 

 

커낵션 수

다음은 커낵션수에 대한 모니터링 화면이다. 커낵션 수의 모니터링은 RDServer의 커낵션 풀을 쓸때만 보여진다.

 

 

상태 블록 스택

오른쪽의 블록 스택은 현재 사용중인 커낵션수를 실시간으로 블록을 이용하여 시각적으로 보여준다.

커낵션수가 25개나 그 이하일 경우에는 노란색 블록이 한 개의 커낵션을 나타내고 그 이상이면 오렌지색 블록이 10개의 커낵션수를 나타낸다. 그래프선의 색깔은 25개나 그 이하는 빨간색으로 25개 이상은 파란색으로 현재 사용중인 커낵션수의 개수를 나타낸다. 상태 블록 스택의 하위에는 정확한 커낵션의 숫자가 표시된다.

상태 블록 스택의 상단에는 커낵션수의 단위와 허용 커낵션 수가 표시된다. 허용 커낵션수는 db.properties에 정의된 최대 커낵션의 수를 표시해준다. 이 커낵션수를 넘으면 커낵션이 더 이상 생성되지 않는다.

 

그래프

왼쪽 상단에는 현재 리포팅중인 서비스명이 나타난다.

X축은 시간의 흐름을 나타낸다. 1초에 한번씩 데이터를 업데이트한다.

Y축은 현재 사용중인 커낵션의 수를 나타낸다. 그 단위는 오른쪽 상단에 있는 레벨에서 확인할 수 있다. 기본적인 단위는 일 단위이다. 25개 이상일 경우는 십 단위로 바뀌어 표시된다.

 

서비스 파일

 

 

상단의 “서비스중인 파일 리스트”의 옆에는 현재 리포팅이 되는 파일의 개수를 나타낸다.

데이터 표시 화면에서는 접속한 시간, 접속한 IP 그리고 파일명을 리스트로 표시한다.

 

전체표시 기능

상단의 모니터링 주메뉴에서 전체표시를 선택하면 모든 모니터링을 한번에 활성화 시킬 수가 있다.

그림 (1) 타일식 정렬

정렬옵션

모니터링의 바탕화면에서 오른쪽 클릭을 하면 정렬옵션 창이 뜬다.

전체표시를 했을 때 기본적으로 정렬되는 방식은 타일식이다. 사용자가 선택하기에 따라 모니터링 창들의 정렬을 바꿀수 있다.
